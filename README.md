# ⚾ MLB 台灣球員動態與賽季數據自動化追蹤系統 (GAS)

這是一個基於 Google Apps Script (GAS) 開發的自動化管理系統，旨在將 Google 試算表與 MLB / MiLB 官方數據 API 進行連動。

涵蓋所有大聯盟與小聯盟球員，
系統能自動爬取並統計旅美台灣球員的最新狀態、近期異動與近五日出賽數據變化，並具備雙核取方塊的「郵件寄送管理中心」，可定時或手動將精美的 HTML 戰報群發給指定訂閱用戶。

## 🚀 核心功能特色

1. **自訂工具列選單 (`onOpen`)**
   * 自動於 Google 試算表上方注入「⚾️ MLB工具」選單，操作直覺。
2. **多維度資料爬蟲 (`UrlFetchApp`)**
   * **基本資訊與異動：** 自動追蹤球員的守備位置、年齡、現狀、所屬球隊、層級，並針對近 5 天內的最新異動自動標記紅色高亮。
   * **賽季數據統計：** 區分 `Pitcher`（投手）與 `Batter`（打者）分頁，動態比對當季累積數據與近五日數據變化（Delta 值），出賽狀態與數據升降一目了然。
3. **智慧防錯與動態驗證 (`onEdit`)**
   * 即時檢查「郵件寄送管理中心」輸入的 Email，自動修正常見拼字錯誤（例如 `@gamail.com`、`@gamil.com` 自動校正為 `@gmail.com`）。
   * 格式錯誤時會自動高亮底色並阻斷發信，防止垃圾資料損害信譽。
4. **雙核取方塊郵件管理中心**
   * 支援「⏰ 每日自動」與「⚡ 立即寄送」雙軌發信機制。
   * 管理者獨有「審查模式」與「中斷攔截通知」：若名單內包含格式錯誤或查無收件人，系統會安全攔截並寄信警告管理者，確保群組發送安全。
5. **高效排程管理**
   * 內建時間驅動觸發器（Trigger），一鍵啟用或關閉每日早上 7 點的自動更新與發信任務。

## 📂 試算表結構要求

為確保程式碼正常運行，您的 Google 試算表必須包含以下特定名稱的分頁（工作表）：

* **`台灣球員清單`**：欄位需包含 `英文名字`、`Player ID`、`MiLB頁面`、`MLB頁面`、`姓名`、`異動日期`、`異動內容`、`球隊層級` 等。
* **`Pitcher`**：投手數據頁。基礎與 Delta 數據欄位包含 `W`, `L`, `ERA`, `G`, `GS`, `SV`, `IP`, `SO`, `WHIP`。
* **`Batter`**：打者數據頁。基礎與 Delta 數據欄位包含 `AB`, `R`, `H`, `HR`, `RBI`, `SB`, `AVG`, `OBP`, `OPS`。
* **`郵件寄送管理中心`**：(可透過功能選單一鍵初始化建置)
* **`Stats_History`** / **`Log`**：系統底層自動初始化，用於暫存歷史數據以計算 Delta 值及紀錄運行日誌（可手動隱藏）。

## 🛠️ 使用與安裝教學

### 第一步：在試算表中部署程式碼
1. 開啟您的 Google 試算表。
2. 點選上方工具列的 **擴充功能 (Extensions)** -> **Apps Script**。
3. 將本專案的 `code.js` 程式碼完整複製並貼入編輯器中，點選儲存。
4. 重新整理 Google 試算表網頁，上方將會出現 `⚾️ MLB工具` 選單。

### 第二步：初始化管理中心
1. 點選 `⚾️ MLB工具` -> `⚙️ 建立/重設【郵件寄送管理中心】`。
2. 系統會自動建立分頁，並填入您的管理者信箱（預設為目前的使用者帳號）。
3. 在欄位中依序填入收件人姓名、信箱，並勾選對應的發信模式。

### 第三步：啟用自動排程
* 點選 `⚾️ MLB工具` -> `⏰ 啟用：每日早上 7 點自動更新與發信`。
* *提示：首次執行時，Google 會跳出權限驗證視窗，請依提示授予 `UrlFetchApp` (連外網路) 與 `MailApp` (發送郵件) 的權限。*

## ⚠️ 未來使用與維護提醒 (Important Notices)

當您未來需要修改介面、優化功能或提供給他人使用時，請務必注意以下關鍵事項：

1. **MLB/MiLB 官方網頁結構變更風險**
   * 程式碼中的 `scrapeMiLBPage` 與 `scrapeDetailedStats` 函數是透過**正則表達式 (Regex)** 直接拆解官方網頁的 HTML 結構。如果未來 MLB 官方更換網站前端架構、改版或變更 CSS Class 名稱，爬蟲將會失效（ Log 會出現錯誤或欄位顯示“未找到”）。
   * **優化建議：** 未來若失效，需進場修正 RegExp 匹配規則，或全面改為對接 MLB 官方的 `statsapi.mlb.com` RESTful API 來撈取更穩定的 JSON 資料。

2. **Google 每日發信配額限制 (Quota Limits)**
   * 本系統使用 `MailApp.sendEmail` 發信。一般 Google 帳戶（免費版）每日發信上限為 **100 封**，Google Workspace 商業版帳戶每日上限為 **1500 封**。
   * 如果您的訂閱用戶群體龐大，密件副本 (`bcc`) 內的信箱數量過多可能導致發信失敗。未來擴充時若超出配額，應考慮串接外部電商電子郵件服務（如 SendGrid, Mailgun）或改用分批發送機制。

3. **API 呼叫頻率與 `Utilities.sleep()`**
   * 為了避免在短時間內對 MLB 伺服器發出過多請求而被阻斷（或被視為惡意攻擊），程式碼在遍歷球員名單時加入了調用延遲（例如 `Utilities.sleep(300)` 與 `Utilities.sleep(400)`）。
   * **維護提醒：** 未來若球員清單大幅擴大，請確保整體執行時間不會超過 GAS 的單次執行時間限制（免費帳戶單次執行上限為 **6 分鐘**）。若超時，需將架構修改為「分批/分段執行」。

4. **歷史數據比對依賴度**
   * 近五日數據變化（Delta）高度依賴 `Stats_History` 工作表內的 JSON 紀錄。若是手動清空該工作表，系統將會需要重新累積數日才能再次正確計算出數據變化數值（期間會顯示 `累積中...`）。
